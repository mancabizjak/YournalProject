BRANCHI
-------
- za razlicne faze razvoja (developed, stable, ...)
- na gitu za vsak feature lahko, ker so cheap, na koncu ga das v master
- comitti ti kazejo snapshote fajlov in pointer na prejsnji komit
- branch = pointer na tvoj commit
-> kako ves na katerem branchu si? (kako git ve)
	- git ima implementiran kazalec head = pointer, ki kaze kdo je na katerem branchu

git checkout -b 'novbranch' -> -b pomeni da ustvaris nov branch in checkoutas
git branch -> na kater branch kaze head
git checkout master -> samo zamenja branch

git log --pretty=oneline
git merge 'novbranch' <- v master doda commit iz druzga brancha
